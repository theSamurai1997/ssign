# `results/` — intermediate and final analysis outputs

This directory holds outputs generated by analyses run for the ssign paper —
things like benchmark tables, enrichment results, cross-genome comparisons.
Raw reference databases go in `data/`, not here.

## What goes here

- Outputs from running the pipeline on the paper's validation/benchmark
  genomes (as TSVs, CSVs, JSON).
- Summary statistics, enrichment tables, cross-validation numbers.
- Anything needed by scripts in `figures/` to regenerate a figure.
- Per-phase subdirectories are fine (`results/benchmark/`,
  `results/validation/`, `results/enrichment/`).

## What must **not** go here

- Raw reference databases — those live in `data/` (or, more accurately, are
  fetched from Zenodo).
- User-generated output from an individual ssign run — those live in
  whatever output directory the user chose.
- Large files over ~50 MB — track those via Zenodo + document them here with
  checksums, don't commit them to git.

## Reproducing

Every meaningful result file should have a companion note explaining how it
was generated — either inline in this README or as a separate file alongside
the result. The rule: someone reading the paper should be able to trace any
number in a figure back to a specific `scripts/` invocation.
